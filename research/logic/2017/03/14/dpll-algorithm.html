Abstract DPLL

Before simulating how DPLL works over the example shown above. I would like to use the abstract version of DPLL which represented in Leonardo de Moura’s slides, Summer School on Logic and Theorem Proving

Each state of the procedure is a 3-tuple consisting of partial assignment , the input clause set and a set of conflict clauses derived from . I will give an intuitive explanation of each rule:

(Decide): select an literal in which has not been assigned yet, then assign it in . It is similar to ramdomly assignment in the example.
(UnitPropagate): with the current assignment , we are looking for some formula in and such that only satisfies this formula if is assigned. It looks like in Sudoku game. You may have some assumptions over some places. The other values over other places are filled based on those assumptions.
(Conflict): The current $M$ leads to conflict state where there is exists clauses such that they are unsatisfiable.
(Resolve): Intuitively, if there is a model satisfies , this model also satisfies .
(Learn): We may imply new formulas after resolving step. Learn simply update new formulas into input set.
(Backjump): Backjump to the point where we can satisfy by new decide.
(Unsat): Obviously, if we unable to find an assignment to resolve conflict, it is unsatisfiable.
