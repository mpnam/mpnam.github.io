    My Favourite React-Native Project Starter

    There are many ways to create new react-native project. This post shows how I init new project in react-native.

    react-native init

    From the React-Native official docs, you can use the command line react-native init <project-name> to generate new project. However, this step will set up your project to work with the lasted version of react-native. It sounds good, but in my experience, the lasted version of react-native is not always compatible with all libraries which you are intend to use later. This failure costs me a lot of time when I first learned react-native. So far, up to this time when I write this post, I rather use react-native version 0.44.0 since it is compatible to almost libraries which I want to use in almost projects. Hence, my recommended command is react-native init <project-name> --version react-native@0.44.0.

    My favourite libraries

    The thing I like most about javascript in general and react-native in particular is there are many packages for many things. Sometimes your job is only using these packages and build your applications without programming too much :-). And then I will list out some packages which I often use:

    react-native-config

    I always have some config variables that I need to access anywhere in my application. With this package, you only need to define your config variable in .env file in the root of your React Native app. For example,

    SITE=https://mpnam.github.io/

    Then you can access variable SITE from anywhere in your app:
