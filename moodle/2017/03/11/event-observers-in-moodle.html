Event Observers in Moodle

Posted in:
  • Moodle

It is essential to support such thing likes event observers in any framework. In this post, I will show how to defined your observers in Moodle.

Events

In Moodle, when an action takes place, an event is created by a core API or plugin. For example, core API events are stored in /lib/classes/event. All events must be extended from abstact class base. In order to trigger an event $event, you simply executing $event->trigger(). For example, the following codes is for trigger the event of adding new blog post.

Observers

Observers are described in /lib/db/events.php in the array $observers. Each observer is defined with the following properties (as described in Moodle’s docs)

    * eventname – fully qualified event class name or "*" indicating all events, ex.: \plugintype_pluginname\event\something_happened.
    * callback - PHP callable type.
    * includefile - optional. File to be included before calling the observer. Path relative to dirroot.
    * priority - optional. Defaults to 0. Observers with higher priority are notified first.
    * internal - optional. Defaults to true. Non-internal observers are not called during database transactions, but instead after a successful commit of the transaction
